# 复杂度评估规则与分解模板

## 复杂度评分细则

### 算法复杂度

| 分数 | 标准 | 示例 |
|--|--|--|
| 1-2 | 简单 CRUD，无业务计算 | 增删改查 |
| 3-4 | 简单计算/转换 | 数据格式转换、简单聚合 |
| 5-6 | 规则引擎/状态机 | 订单状态流转、审批流程 |
| 7-8 | 复杂算法 | 调度算法、匹配算法 |
| 9-10 | 研究级算法 | 机器学习推理、复杂优化 |

### 数据复杂度

| 分数 | 标准 | 示例 |
|--|--|--|
| 1-2 | 单表 < 1 万行 | 配置表、字典表 |
| 3-4 | 多表 < 10 万行 | 用户管理、订单管理 |
| 5-6 | 多表 < 百万行，有关联查询 | 报表系统、历史数据 |
| 7-8 | 大数据量 > 百万，需分片 | 日志分析、行为追踪 |
| 9-10 | 实时流处理 / 分布式数据 | 实时推荐、流式计算 |

### 集成复杂度

| 分数 | 标准 | 示例 |
|--|--|--|
| 1-2 | 无外部依赖 | 纯本地功能 |
| 3-4 | 1-2 个稳定依赖 | 调用内部微服务 |
| 5-6 | 3+ 依赖，部分不稳定 | 调用银行接口、短信网关 |
| 7-8 | 多个异步依赖需编排 | Saga 模式、事件编排 |
| 9-10 | 跨平台/跨网络集成 | 硬件集成、遗留系统对接 |

### 并发复杂度

| 分数 | 标准 | 示例 |
|--|--|--|
| 1-2 | 无并发竞争 | 单用户操作 |
| 3-4 | 简单乐观锁 | 库存扣减、版本控制 |
| 5-6 | 需要分布式锁 | 全局唯一流水号 |
| 7-8 | 复杂的并发编排 | 多阶段提交、分布式事务 |
| 9-10 | 高性能无锁方案 | 无锁队列、Actor 模型 |

### 领域复杂度

| 分数 | 标准 | 示例 |
|--|--|--|
| 1-2 | 规则简单明确 | 表单提交 |
| 3-4 | 多分支逻辑 | 条件审批、权限判断 |
| 5-6 | 复杂业务规则 + 异常场景 | 计费规则、合同管理 |
| 7-8 | 领域专家才能理解的规则 | 医疗诊断、金融风控 |
| 9-10 | 法规合规 + 持续变化 | 税务计算、跨境结算 |

## 分解模板

### 垂直切片模板

```markdown
## 功能: [名称]

### 切片 1: MVP (优先级 P0)
- 核心用户故事: [...]
- 范围: 仅 happy path
- 技术: 最简单的实现方案

### 切片 2: 完善 (优先级 P1)
- 异常处理 + 边界场景
- 性能优化

### 切片 3: 增强 (优先级 P2)
- 高级功能
- 可扩展性改造
```

### 水平分层模板

```markdown
## 模块: [名称]

### Step 1: 接口定义
- 定义 API 契约 + DTO
- 可 Mock 集成

### Step 2: 领域层
- Entity + 业务逻辑 + 单元测试

### Step 3: 持久化层
- Repository + Mapper + 集成测试

### Step 4: 应用层
- Service 编排 + 事务 + API 测试
```

## 估时指南

| 复杂度 | 估时乘数 | 说明 |
|--|--|--|
| 1-3 | 1x | 正常估时 |
| 4-6 | 1.5x | 加 50% Buffer |
| 7-8 | 2x | 加 100% Buffer + Spike |
| 9-10 | 3x+ | 需要原型验证后再估时 |
