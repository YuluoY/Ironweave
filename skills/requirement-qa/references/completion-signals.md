# 完成信号

## 允许结束的信号

以下文本命中任一项时，可视为用户明确允许结束 QA 并输出摘要：

- `开始产出`
- `开始输出`
- `可以写文档了`
- `可以写代码了`
- `按这个做`
- `按这个输出`
- `整理成 PRD`
- `进入实现`
- `开始执行`
- `start implementation`
- `generate now`
- `够了，开始吧`

## 继续意图保护

如果回答中出现以下语义，优先视为"继续澄清"，不要误判结束：

- `继续问`
- `继续澄清`
- `先别开始`
- `还不要开始`
- `最后我会说开始产出`
- `等我说可以再开始`
- `还有一个问题`

## 最低结束标准

即使出现完成信号，也应确保以下信息已大致明确：

1. **最终产物**：要交付什么
2. **目标用户 / 验收者**：给谁用
3. **范围边界**：做什么 + 不做什么
4. **关键约束**：时间、技术、资源限制
5. **成功标准**：做到什么程度算完成

如果用户坚持立即开始但信息仍不完整，应在最终产出中**显式列出假设**，标注为"未经用户确认，基于推断"。
