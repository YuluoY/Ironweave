# 难度评分细则（1-10 分制）

## 维度评分标准

### 范围 (Scope) — 权重 30%

| 分数 | 标准 |
|------|------|
| 1 | 单文件内的微修改（≤ 5 行，纯文案/数值） |
| 2 | 单文件局部修改（≤ 20 行） |
| 3 | 单文件较大修改 或 2-3 个紧密相关文件的小改动 |
| 4 | 单模块内 3-5 个文件 |
| 5 | 单模块内 5-8 个文件，或跨 2 个模块各少量文件 |
| 6 | 跨 2-3 个模块，涉及 8-12 个文件 |
| 7 | 跨 3-4 个模块，涉及 12-20 个文件 |
| 8 | 跨 4+ 个模块，涉及 20+ 文件 |
| 9 | 多数模块受影响，可能需要批量修改 |
| 10 | 全局性变更，几乎所有模块都需修改 |

### 深度 (Depth) — 权重 25%

| 分数 | 标准 |
|------|------|
| 1 | 无逻辑变更（纯文案替换） |
| 2 | 极简逻辑（配置值修改、样式调整） |
| 3 | 简单逻辑（条件分支、字段映射） |
| 4 | CRUD 模板化操作（增删改查页面，套路化） |
| 5 | 中等逻辑（表单校验、API 集成、组件状态管理） |
| 6 | 较复杂逻辑（权限控制、工作流、乐观更新） |
| 7 | 复杂逻辑（状态机、并发处理、缓存策略） |
| 8 | 高度复杂（自定义构建流程、性能关键路径优化） |
| 9 | 算法密集（复杂数据结构、分布式协调） |
| 10 | 系统级复杂度（编译器/解释器、实时计算引擎） |

### 耦合 (Coupling) — 权重 20%

| 分数 | 标准 |
|------|------|
| 1 | 完全独立，无外部依赖 |
| 2 | 使用外部模块但不修改其接口 |
| 3 | 修改模块内部实现，接口不变 |
| 4 | 修改模块接口，已知调用方 ≤ 3 处 |
| 5 | 修改模块接口，调用方 3-5 处 |
| 6 | 修改公共接口，调用方 5-10 处 |
| 7 | 修改公共接口，调用链 > 3 层 |
| 8 | 修改基础设施组件（路由、中间件） |
| 9 | 修改核心抽象（ORM 基类、通用组件库） |
| 10 | 修改底层框架/运行时，影响全局所有模块 |

### 风险 (Risk) — 权重 15%

| 分数 | 标准 |
|------|------|
| 1 | 完全可逆，出错无影响 |
| 2 | 可逆，出错影响极小（开发环境配置） |
| 3 | 可逆但需额外工作（功能回退） |
| 4 | 回退成本中等（需要测试验证） |
| 5 | 部分不可逆（数据库 Schema 有 down migration） |
| 6 | 回退困难（生产配置变更、第三方 API 对接） |
| 7 | 涉及用户数据格式变更 |
| 8 | 涉及安全策略或加密方案 |
| 9 | 涉及付款/金融/合规流程 |
| 10 | 不可逆且高危（删除生产数据、密钥轮换） |

### 认知 (Cognition) — 权重 10%

| 分数 | 标准 |
|------|------|
| 1 | 纯机械操作，无需任何领域知识 |
| 2 | 需了解单个配置项或 API |
| 3 | 需了解一个技术概念的基本用法 |
| 4 | 需理解一个完整的技术方案（如 JWT 认证流程） |
| 5 | 需理解业务领域规则（权限模型、计费逻辑） |
| 6 | 需理解复杂业务流程（工作流、审批链） |
| 7 | 需要跨领域知识（前端+后端+数据库） |
| 8 | 需要跨领域+业务（技术方案+安全+业务合规） |
| 9 | 需要专家级单领域知识（密码学、分布式一致性） |
| 10 | 需要多个专家级领域知识（编译+安全+行业法规） |

## 加权平均 → 等级映射

```
加权分 = Scope×0.30 + Depth×0.25 + Coupling×0.20 + Risk×0.15 + Cognition×0.10
```

| 加权分范围 | 等级 | 名称 |
|-----------|------|------|
| 1.0 - 2.0 | L1 | 琐碎 |
| 2.1 - 4.0 | L2 | 简单 |
| 4.1 - 6.0 | L3 | 中等 |
| 6.1 - 8.0 | L4 | 复杂 |
| 8.1 - 10.0 | L5 | 重大 |

## 偏置上浮规则

不再整级上浮，改为**加权分 +0.5 偏置**：

```
IF 范围 ≤ 2 AND 深度 ≤ 2 AND 耦合 ≤ 2 AND 风险 ≤ 3:
    → 豁免偏置（明确标注理由）
ELSE:
    → 加权分 += 0.5（上限 10.0）
```

**效果示例**：

| 原始加权 | 原始等级 | +0.5 后 | 最终等级 | 是否跨级 |
|---------|---------|--------|---------|---------|
| 1.5 | L1 | 2.0 | L1 | 否 |
| 1.9 | L1 | 2.4 | L2 | 是（边界） |
| 3.2 | L2 | 3.7 | L2 | 否 |
| 3.8 | L2 | 4.3 | L3 | 是（边界） |
| 5.5 | L3 | 6.0 | L3 | 否 |

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 任务描述模糊（< 20 字） | 认知 +1，范围 +1 |
| 涉及遗留代码 | 耦合 +1 |
| 首次接触的模块 | 认知 +1 |
| 用户明确说"简单" | 不因此降分，按实际评估 |
| 无法获取项目上下文 | 所有维度下限 +1 |
